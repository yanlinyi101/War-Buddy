# 完整玩法愿景与核心循环

日期：2026-04-26
项目：War Buddy（Godot 4.6.x）
状态：草案。MVP 之后玩法的北极星愿景文档，是 07–12 的母文档。

> 本文为 [`06-full-gameplay-vision.md`](06-full-gameplay-vision.md) 的中文参考版。两份内容保持同步；如发生冲突，以英文版为权威。

## 1. 目的

定义 War Buddy 长期想成为什么样的游戏，让所有后续设计决策（命令系统、AI 副官架构、实体规则、备战界面、MVP 手感调参）都能回溯到一组统一的产品支柱。

本文是**北极星**，不是实施合同。这里刻意不写具体数值、兵种表、schema——这些放在 07–12。本文锁定的是：*这是什么类型的游戏*、*它和别的同类游戏不一样在哪*。

01–05 描述 Godot MVP 切片。06 开启 MVP 之后的设计线。

## 2. 产品 DNA

下面五条支柱是承重墙。任何一条变了，都会让 07–12 的部分章节失效，必须重新 brainstorm，不能私下编辑。

### 2.1 指挥官在场，不是全知操作员

玩家是身处战场的指挥官，亲手操控**一个**英雄单位。玩家**不是**俯视全图、框 50 个农民、排 12 个建筑队列的传统 RTS 操作员。

**含义**：玩家用来影响己方阵营其它部分的输入，是"对副官下命令"，不是"亲手微操单位"。多选、拖框、编队键、生产队列都明确不是主输入方式。

### 2.2 真正的 LLM 驱动 AI 副官，是主输入通道

玩家通过**单一 AI 副官**——agent 阶梯（§2.3）顶端的指挥级官员——来调动阵营其它部分，**语音是主输入**，**真 LLM 推理**是执行后端。副官把专业化工作（战斗节奏、经济维护、侦察轮换、防御反应）派发给下层小队长——专业化是小队长的属性，不是切分成独立副官席位。文本输入仅作为 MVP 阶段的验证通道，不在最终上线产品里出现。

**含义**：LLM 推理的延迟、成本、不可预测性，都是系统第一类约束。架构必须吸收它们，不能假装它们不存在。

### 2.3 Agency 是梯度：英雄 → 副官 → 小队长 → 普通兵

阵营不是"玩家 + AI 工具"。它是一个四级 agency 梯度，玩家的情感投入随之递减：

| 层级 | Agency | 实体形态 | 玩家关系 | 死亡处理 |
|---|---|---|---|---|
| **英雄** | 玩家直接控制 | 场上单位 | 玩家化身 | Ragdoll 尸体 + 漂浮灵魂 |
| **副官** | 顶级 LLM agent | **场外**——HUD 头像 + 语音，无可攻击实体 | 长期羁绊，跨局完整记忆 | **不会阵亡，不可被攻击** |
| **小队长** | 较小的 LLM agent | 场上单位 | 跨局记忆 + 适度属性强化（每轴 ≤15%） | Ragdoll 尸体 + 漂浮灵魂 |
| **普通兵** | 行为树脚本，无 agency | 场上单位 | 一次性 | 仅 ragdoll 尸体 |

副官如果对局间不记得玩家——支柱破了；副官如果从不犯错（战略误判、误处理小队长汇报、压力下选错预案）——支柱也破了：玩家无法和神谕产生羁绊。**失败面是认知层，不是物理层**：副官在场外指挥，不能被击杀。玩家感受到的风险是"我的副官此刻是否真的理解我"，不是"我的副官会不会被集火"。

小队长是故意做**小**的设计，不是骑砍式家臣/朝堂子系统：其全部机械面只有**持久记忆 + 每轴 ≤15% 属性强化 + 可阵亡**三件套。玩家羁绊来自这三个机制本身——没有招募任务、没有忠诚度政治、没有联姻树。保持小，剩下的让 LLM 人格本身去做。

**含义**：08 号文档涵盖**整条 agent 阶梯**，不只是副官。09 号必须给每条单位定义带一个 `agency_tier` 字段。11 号根据 agency 层级决定灵魂特效等视觉反馈是否触发——并明确把副官排除在场上碰撞与尸体管线之外。

### 2.4 三层命令系统

命令在三个层进入系统，每层走不同的执行路径：

| 层 | 输入位置 | 执行路径 | 延迟 | 可预测性 |
|---|---|---|---|---|
| **预案**（战术预案） | 局前备战 UI | 结构化 schema → 行为树 | 即时 | 完全确定 |
| **战术指令** | 局内语音/文本 | 意图分类器 → 行为树 | 低 | 高 |
| **战略意图** | 局内语音 | LLM 规划器 → 拆成战术指令 → 行为树 | 高 | LLM 边界内 |

三层共享**同一份"战术指令" schema** 作为公共底层——预案直接序列化成它、LLM 规划器输出它、行为树消费它。这份共享 schema 是 07 号文档最重要的产物。

**指挥链严格（A 链）。** 局内所有命令的收件人都是**副官**。副官将复杂命令分解为针对各小队长的指令，依然走同一份战术 schema——这让"副官 ↔ 小队长"通道成为结构同构的内层循环。预案在备战界面里以明确的角色 / 小队长 / 网格引用编写；运行时由玩家对副官触发，副官分发给麾下小队长。指挥官局内**永不**绕过副官。小队长不接受玩家直接命令；它们与玩家的关系活在持久记忆和局后结算里（§2.3），不在战术寻址通道。

### 2.5 副官混合模式：AI 主，人类执政官副

阵营有**一个**副官席位——指挥级官员。默认填充 AI 副官。作为补充玩法，人类玩家可以接管该席位，类似星际2 执政官（Archon）模式——和指挥官共享一个阵营的控制权。**人类副官模式是一个特性，不是 AI 坏了的兜底**。

小队长（见 2.3）是另一档更低层的 agent，**不是同类席位**——它们是局内随部队一起出现的战斗单位 agent，不是局前的指挥级角色。

**含义**：命令总线必须能同时接受 AI handler 和人类输入，不能为此分叉游戏其它逻辑。

## 3. 核心游戏循环

一场对局是一个连续循环，不是分阶段：

1. **读战场**——指挥官通过摄像机直观察 + 小地图 + 副官汇报
2. **形成意图**——指挥官决定下一步要发生什么
3. **下命令**——选择合适的层（预案触发 / 战术 / 战略）
4. **副官执行**——按层不同的自主度推进
5. **指挥官并行亲手操作**——用英雄打架、侦察、争抢关键点
6. **接收反馈**——副官回报状态、完成、问题、需要确认
7. **调整**

英雄单位让指挥官在副官处理宏观时仍然有事干。没有英雄玩家就成了观众；没有副官玩家就退化成传统 RTS 微操工。两半都不能少。

## 4. 空间词汇表

地图地址用三层（schema 全文在 07）：

- **网格**——每张地图共享的 A1–H8（或类似）网格 overlay。预案 schema 和精确指令使用。
- **设计师地标**——关卡作者放置的命名特征（如"北矿"、"B4 高地"）。是网格的别名。
- **玩家自定义命名区域**——备战界面里的进阶解锁功能。玩家自己给区域命名，AI 副官学习这套私人词汇表。是**可解锁深度**，不是基础设施。

## 5. 比赛形态（延后）

比赛长度、阵营数、胜利条件、不对称性、单人/PvP/PvAI 等——都**有意暂不写死**。它们受第 2 节五条支柱约束，但具体值留到 09 号文档拍板（在那时实体和经济规则也开始落地）。

愿景层确定的只有：*无论一场比赛长什么样，指挥官永远控英雄、永远至少有一名副官、永远通过三层命令系统下令。*

## 6. 范围内 / 范围外

### 范围内（在 07–12 里增量构建）

- 一名指挥官 + 一个英雄 + 一个副官席位（AI 或人类，场外）
- 小队长作为局内较小的 LLM agent，机械面仅含持久记忆 + ≤15% 属性强化 + 可阵亡——不再扩展家臣机制
- 部队规模约 50 单位/阵营（1 英雄 + N 小队长 + 普通兵；副官在场外，不计入）
- 语音作为主命令通道
- 三层命令系统 + 共享战术 schema
- 跨对局持久化的副官身份；小队长记忆深度由测试调整
- 局前备战 UI + 预案编辑器
- 玩家自定义命名区域（进阶）
- 足够让副官与小队长都有自主工作可干的经济与战斗单位

### 明确范围外（违反 DNA，未重新打开 06 不得提议）

- 玩家侧多选、拖框、编队组
- 玩家直接管理生产队列
- 指挥官框选 / 微操任意单位（仅英雄被直接控制）
- 没有持久身份的 AI 副官（"每局都忘了玩家"）
- 把 agent 梯度坍缩回单一档（副官 = 小队长 = 普通兵）
- 指挥官局内绕过副官直接对小队长下令（必须始终经由副官）
- §2.3 三件套之外的家臣 / 朝堂 / 忠诚度 / 招募系统
- 副官作为场上可被攻击实体（副官不会阵亡、不可被集火、无 HP 条）
- 上线产品中用快捷键命令面板替代语音
- 上线产品中以脚本/有限状态机伪装成 AI 副官（仅 MVP 阶段允许）

### 暂时延后（后续文档可能重新打开）

- 联网 PvP、匹配、天梯
- Mod / 自定义副官创作
- 非地球 RTS 题材、剧情战役
